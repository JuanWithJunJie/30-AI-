# 引导式小红书 AI 术语科普营 · Guided Skill

本 Skill 在生产卡片前，先通过 **3 轮提问** 收集关键参数，再按用户确认的内容定制生成。
所有生产规范与原版 `xiaohongshu-ai-campaign` 完全一致，区别仅在于：
- 受众岗位由用户自定义，不固定为「管理层/工程师/品质IE」
- 行业背景可切换（制造业/互联网/医疗/金融/教育等）
- 生产前先确认方案，用户可修改后再执行

---

## 第一步：启动引导流程

**收到任务后，不要直接生产内容。先执行以下引导流程。**

### 🔴 第一轮提问 — 本期主题

向用户展示以下问题（使用 `ask_user_input_v0` 工具展示选项，或直接提问）：

```
📋 引导问题 1/3 — 本期主题

请告诉我这期要讲哪个 AI 术语？
可以直接说术语名（如「RAG」「幻觉」「Agent」），
或描述一个你想解释的概念/场景。

如果你不确定选哪个，我可以根据你的受众推荐。
```

收到回答后，确认理解并进入第二轮。

---

### 🔴 第二轮提问 — 目标受众与岗位

```
📋 引导问题 2/3 — 目标受众

这期内容面向哪些岗位的同事？请列出 2～4 个岗位。

【常见示例】
研发类：硬件开发 / 软件开发 / 嵌入式工程师 / 算法工程师
质量类：测试工程师 / QA / 可靠性工程师
业务类：产品经理 / 项目经理 / 运营 / 销售
管理类：技术总监 / 部门经理 / CTO
制造类：工艺工程师 / 设备工程师 / 品质主管

你可以直接告诉我，比如：
「硬件开发、软件开发、测试工程师、产品经理」
```

收到岗位列表后，进入第三轮。

---

### 🔴 第三轮提问 — 行业背景与场景偏好

```
📋 引导问题 3/3 — 行业与场景

最后两个小问题：

① 你们是哪个行业？（这样我能选合适的举例场景）
   例：消费电子 / 汽车 / 互联网 / 医疗器械 / 半导体 / 金融科技 / 教育…

② 有没有特别想突出的应用场景或避开的内容？
   例：「想多讲代码层面」「不要太技术」「结合我们产品的质量管控流程」
   没有特殊要求可以直接说「随你」
```

---

### ✅ 确认方案

收集完三轮答案后，生成一份方案摘要给用户确认，格式如下：

```
📝 本期方案确认

术语主题：[用户填写的术语]
目标受众：[岗位1] / [岗位2] / [岗位3] …
行业背景：[行业]
特殊要求：[有/无]

卡片结构规划：
  卡片1：封面
  卡片2：[岗位1] 视角故事
  卡片3：[岗位2] 视角故事（深色技术感背景）
  卡片4：概念解释（通俗类比）
  卡片5：[行业] 场景下的 3 个实战案例
  卡片6：误区与选型技巧
  卡片7：总结 + 互动问题

---
确认无误可以开始生产，或告诉我需要调整的地方。
```

**等待用户确认后，再开始生产。**

---

## 第二步：按岗位定制内容规则

用户确认方案后，严格按以下规则映射岗位到卡片内容：

### 岗位视角映射表

根据用户给出的岗位，按以下逻辑分配卡片2、3（其余卡片结构固定）：

| 岗位类型 | 卡片2（偏业务/管理） | 卡片3（偏技术/实现） |
|----------|------------------|------------------|
| 硬件开发 | 硬件架构师 / 项目负责人视角：成本、选型、集成挑战 | 硬件工程师视角：电路原理、芯片架构、信号处理 |
| 软件开发 | 技术负责人 / 架构师视角：系统设计、技术债、落地路径 | 软件工程师视角：代码实现、API 调用、框架选型 |
| 测试工程师 | QA 主管视角：测试覆盖率、质量门禁、风险把控 | 自动化测试工程师视角：测试框架、AI 辅助测试工具 |
| 产品经理 | 产品总监视角：功能规划、竞品分析、用户价值 | 产品经理视角：需求拆解、AI 能力边界、原型设计 |
| 算法工程师 | AI 团队负责人视角：项目立项、资源申请、ROI 评估 | 算法工程师视角：模型原理、训练细节、推理优化 |
| 运营 | 运营总监视角：数据指标、用户增长、效率提升 | 运营工程师视角：自动化工具、数据分析、A/B 测试 |
| 设备工程师 | 设备主管视角：OEE、停机代价、维护策略 | 设备工程师视角：传感器、信号采集、预测模型 |
| 工艺工程师 | 工艺经理视角：良率目标、参数管控、品质成本 | 工艺工程师视角：实验设计、参数关联、根因分析 |

**当用户给出 2 个岗位时**：
- 卡片2 → 第1个岗位（通常选较偏管理/决策的那个）
- 卡片3 → 第2个岗位（通常选较偏技术/执行的那个）

**当用户给出 3～4 个岗位时**：
- 卡片2 → 选最偏业务/管理的岗位
- 卡片3 → 选最偏技术/实现的岗位
- 在卡片5实战场景中，用3个不同场景分别照顾其余岗位的应用视角

**当用户给出 1 个岗位时**：
- 卡片2 → 该岗位的管理者/负责人视角（更宏观）
- 卡片3 → 该岗位的一线执行者视角（更技术）

## 第三步：行业背景场景库

根据用户填写的行业，在卡片5实战场景中优先使用对应行业的举例：

### 行业场景对应

**消费电子 / 手机**
- 场景方向：摄影AI、质检、供应链、客服、多语言

**汽车 / 新能源**
- 场景方向：自动驾驶训练数据、OTA升级、零件预测维护、车控系统测试

**互联网 / 软件**
- 场景方向：AI代码生成、推荐系统、用户行为分析、内容审核、客服机器人

**医疗器械 / 医疗**
- 场景方向：影像辅助诊断、药物研发、病历分析、合规检查、设备预防维护

**半导体 / 芯片**
- 场景方向：晶圆缺陷检测、EDA工具辅助、良率提升、封测自动化

**金融科技**
- 场景方向：风控模型、合规审查、报告生成、智能投顾、客户分层

**教育 / EdTech**
- 场景方向：个性化学习路径、作业批改、内容生成、学情分析

**通用（无特定行业）**
- 使用多行业混合举例，注明「以下为示意性举例」

> ⚠️ 所有案例中的企业/机构描述必须匿名化（「某头部XX企业」），不得写具体公司名。

## 第四步：完整生产流程（与原版相同）

确认方案后，执行以下标准生产流程：

### 必读 Skill

### HTML 模板写入
- 路径：`/home/claude/dayXX_template.html`
- Logo 占位符：`  LOGO_WHITE_PLACEHOLDER` / `  LOGO_BLACK_PLACEHOLDER`

### Logo 注入
```python
with open('/home/claude/logo_final_white_b64.txt') as f: B64W = f.read().strip()
with open('/home/claude/logo_final_black_b64.txt') as f: B64B = f.read().strip()

LW = f'  <div class="card-logo card-logo--white">\n    <img src="data:image/png;base64,{B64W}" alt="制造中心">\n  </div>'
LB = f'  <div class="card-logo card-logo--black">\n    <img src="data:image/png;base64,{B64B}" alt="制造中心">\n  </div>'

html = html.replace('  LOGO_WHITE_PLACEHOLDER', LW)
html = html.replace('  LOGO_BLACK_PLACEHOLDER', LB)
```
