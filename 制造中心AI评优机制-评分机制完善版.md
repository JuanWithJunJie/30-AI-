# 制造中心AI评优机制（评分机制完善版）

## 一、目的

为推动AI在制造中心的规模化落地，强化价值导向与能力沉淀，建立标准化、可量化、可复盘的评优机制，激励员工开展AI应用创新与推广，特制定本机制。

---

## 二、适用范围

- 制造中心全员（含产线、工艺、质量、设备、IT等）
- 所有AI相关项目（AI+硬件 / AI+数据 / AI+系统）
- 以项目为单位参与项目类奖项评审，以个人为单位参与个人类奖项评审

---

## 三、评优原则

### 1. 价值优先
优先奖励对生产、质量、效率、成本、安全产生实际影响的项目。

### 2. Skill沉淀（加分鼓励）
Skill沉淀不作为参评硬门槛，但作为单独加分项纳入评审。对形成高质量Skill卡、标准模板、可复用方法包并已被验证复用的项目，给予显著加分。

### 3. 规模化导向
优先奖励可跨产线、跨车间、跨工厂复制的项目。

### 4. 真实落地
禁止纯方案、PPT或Demo项目以落地类奖项名义参评。

### 5. 证据驱动
所有评分必须有证据支撑，无证据项不得给满分。

---

## 四、奖项设置

### 1. 价值落地奖（核心奖项）
评选对象：已在实际业务场景稳定运行，并形成明确业务价值的项目。

### 2. 创新突破奖
评选对象：具有明显技术探索价值、技术难度高、对后续规模化具有引领作用的项目。允许尚未全面推广，但必须完成可验证试点或原型验证。

### 3. 复制推广奖
评选对象：已形成标准化方法，并在多个部门、产线或工厂复用的项目。

### 4. AI先锋个人
评选对象：在AI项目立项、推动、落地、推广或Skill沉淀中贡献突出的个人。

---

## 五、评优周期

| 周期 | 内容 | 输出 |
| -- | -- | -- |
| 月度 | 快速激励（轻评审） | 优秀项目/个人月度通报 |
| 季度 | 核心评优 | 季度奖项评定 |
| 年度 | 标杆评选 | 年度标杆项目/个人 |

说明：月度以发现与鼓励为主，季度和年度以正式打分与评奖为主。

---

## 六、参评门槛（硬性要求）

### （一）项目类奖项通用门槛
参评项目必须同时满足以下要求：

1. 已完成项目唯一编号登记
2. 已明确项目负责人、责任二级部门、应用场景
3. 已提交项目证明材料
4. 不属于重复拆分申报或历史获奖项目重复申报

### （二）不同奖项附加门槛

#### 1. 价值落地奖
- 已在真实业务场景使用连续不少于2周
- 提供上线证明或现场使用证明
- 至少提供一项价值证明材料（效率、质量、成本、安全等）

#### 2. 创新突破奖
- 已完成PoC、试点或原型验证
- 提供技术验证材料、测试结果或试点结论
- 即使未全面推广，也必须证明技术路径可行

#### 3. 复制推广奖
- 已在至少2个业务单元应用，业务单元包括：部门、产线、车间、工厂中的任一层级
- 提供复用清单、推广记录或复用证明
- 具备可复用文档、模板、培训材料或Skill沉淀的项目优先

#### 4. AI先锋个人
- 至少参与1个合规AI项目
- 其参与项目需具备基本证明材料
- 提供个人贡献说明，并由直属主管确认
- 有方法沉淀、Skill卡输出或团队带教成果者优先

---

## 七、申报材料要求

### （一）项目类奖项申报材料
每个项目至少提交以下材料：

1. 项目基本信息表
2. 项目证明材料
3. 价值证明材料
4. 推广复用材料（如申报复制推广奖）
5. Skill卡、标准模板、操作手册、培训材料等沉淀材料（如有，建议提交，用于加分）

### （二）证据材料清单
可接受的证明材料包括但不限于：

- 系统截图
- 现场照片/视频
- 使用记录
- 业务报表
- 对比数据（改造前/改造后）
- 用户反馈记录
- 培训签到或推广记录
- 会议纪要或验收记录

### （三）无证据限制原则
- 无量化对比数据的价值项，单项最高不得超过该项满分的50%
- 无复用证明的推广项记0分
- 无Skill沉淀材料的项目，Skill加分项记0分，但不影响基础参评资格

---

## 八、评分机制

## （一）项目类奖项评分总则

项目类奖项采用“基础分100分 + Skill沉淀加分15分”的结构。

其中，基础分用于判断项目业务与技术水平，Skill加分用于拉开项目沉淀质量和可推广深度的差距。

基础分包含四个一级维度：

| 维度 | 分值 | 说明 |
| -- | -- | -- |
| 价值贡献 | 40分 | 项目对业务结果的实际影响 |
| 技术与创新 | 20分 | 技术难度、创新程度、方法先进性 |
| 可复制性 | 20分 | 标准化程度、推广复用情况 |
| 项目质量 | 20分 | 资料完整、运行稳定、维护与复盘质量 |

另设：

| 加分项 | 分值 | 说明 |
| -- | -- | -- |
| Skill沉淀加分 | 15分 | 对Skill卡、模板、SOP、培训材料、复用验证等沉淀成果进行加分 |

最终得分 = 基础分 + Skill沉淀加分，满分115分。

说明：不同奖项使用同一套基础评分框架，但设定不同的入围门槛与优先条件。入围先看基础分，获奖排序看最终得分。

---

## （二）项目类奖项详细评分标准

### 1. 价值贡献（40分）

#### 1.1 影响范围（15分）
| 情况 | 分值 |
| -- | -- |
| 仅个人或单岗位使用 | 5分 |
| 覆盖单条产线/单工序 | 10分 |
| 覆盖单工厂多个场景或关键环节 | 15分 |

#### 1.2 效率提升（10分）
| 情况 | 分值 |
| -- | -- |
| 无明确提升或无法证明 | 0分 |
| 提升<10%，或仅有定性反馈 | 3分 |
| 提升10%~30%，有基础对比数据 | 6分 |
| 提升>30%，且有稳定连续数据支撑 | 10分 |

#### 1.3 人工替代/负担降低（5分）
| 情况 | 分值 |
| -- | -- |
| 无明显替代 | 0分 |
| 减少部分重复操作 | 2分 |
| 显著降低人工投入或减少岗位负荷 | 5分 |

#### 1.4 业务结果改善（10分）
| 情况 | 分值 |
| -- | -- |
| 无明确结果改善 | 0分 |
| 对质量/成本/交付/安全有单点改善 | 5分 |
| 对质量/成本/交付/安全形成显著改善且可举证 | 10分 |

评分要求：价值贡献必须基于真实业务场景，不接受仅以“预估价值”作为满分依据。

---

### 2. 技术与创新（20分）

#### 2.1 技术复杂度（10分）
| 情况 | 分值 |
| -- | -- |
| 基于现成工具的简单调用 | 3分 |
| 涉及模型编排、数据处理、流程集成 | 6分 |
| 涉及多系统打通、复杂逻辑或工程化能力 | 10分 |

#### 2.2 创新性（10分）
| 情况 | 分值 |
| -- | -- |
| 行业内常规做法，替代性强 | 3分 |
| 在既有流程上有明显优化创新 | 6分 |
| 在制造场景中具有首创性、探索性或明显示范作用 | 10分 |

评分说明：创新性不等于“概念新”，必须体现为场景方法创新、流程创新或系统集成创新。

---

### 3. 可复制性（20分）

#### 3.1 标准化程度（10分）
| 情况 | 分值 |
| -- | -- |
| 无文档、无模板、无法交接 | 0分 |
| 有基础文档或操作说明 | 5分 |
| 形成标准流程、模板、Skill卡或可直接推广包 | 10分 |

#### 3.2 推广情况（10分）
| 情况 | 分值 |
| -- | -- |
| 未推广 | 0分 |
| 已在1个额外业务单元复用 | 5分 |
| 已在2个及以上业务单元复用 | 10分 |

评分说明：复制推广奖原则上要求本维度不低于14分。

---

### 4. 项目质量（20分）

#### 4.1 资料完整性（5分）
- 项目背景、目标、方案、结果、证明材料齐全：5分
- 材料不完整但能支撑基本评审：3分
- 材料明显缺失：0分

#### 4.2 运行稳定性（5分）
- 连续稳定运行3个月以上：5分
- 连续运行1~3个月：3分
- 仅短期试运行或稳定性不足：1分

#### 4.3 可维护性（5分）
- 有负责人、有维护说明、有异常处理方式：5分
- 部分具备：3分
- 无维护机制：0分

#### 4.4 复盘闭环质量（5分）
- 已形成问题复盘、经验总结、优化建议，闭环清晰：5分
- 有基础复盘记录，但不完整：3分
- 无复盘闭环材料：0分

---

### 5. Skill沉淀加分（15分）

#### 5.1 加分原则
Skill沉淀不是参评门槛，但属于重点鼓励项。对有沉淀、有复用、有推广验证的项目，给予额外加分。

#### 5.2 加分标准
| 情况 | 分值 |
| -- | -- |
| 无Skill卡、无模板、无沉淀材料 | 0分 |
| 有基础Skill卡或文档沉淀，但仅供项目内部使用 | 5分 |
| Skill卡、模板、SOP、培训材料较完整，可供其他团队参考复用 | 10分 |
| 形成高质量沉淀包，且已有跨团队、跨产线或跨工厂复用验证 | 15分 |

评分说明：Skill加分需提交对应材料或复用证明。无材料不得加分。

---

## （三）个人奖评分标准（AI先锋个人，100分制）

| 维度 | 分值 | 说明 |
| -- | -- | -- |
| 项目主导程度 | 40分 | 是否主导项目推进、方案形成与结果交付 |
| 协同推动能力 | 25分 | 是否推动跨部门合作、资源协调、问题闭环 |
| Skill沉淀贡献 | 20分 | 是否推动方法沉淀、知识复用、模板输出 |
| 影响力与带动作用 | 15分 | 是否对团队学习、项目复制、AI氛围建设有明显带动 |

### 个人奖评分口径

#### 1. 项目主导程度（40分）
- 仅执行分配任务：10分以内
- 负责某一模块并独立交付：20~30分
- 主导项目立项、推进、交付、复盘：31~40分

#### 2. 协同推动能力（25分）
- 仅在本部门内配合：10分以内
- 能推动跨岗位协作：11~18分
- 能推动跨部门/跨工厂协同落地：19~25分

#### 3. Skill沉淀贡献（20分）
- 参与Skill整理：5~10分
- 主导输出高质量Skill卡并被复用：11~20分

#### 4. 影响力与带动作用（15分）
- 有基本分享或带教：5分以内
- 能带动周边团队应用AI：6~10分
- 对组织推广有明显示范作用：11~15分

---

## 九、奖项入围与授奖规则

### （一）项目类奖项入围分数线
| 奖项 | 入围条件 |
| -- | -- |
| 价值落地奖 | 基础分≥80分，且价值贡献≥25分 |
| 创新突破奖 | 基础分≥75分，且技术与创新≥14分 |
| 复制推广奖 | 基础分≥80分，且可复制性≥14分 |

### （二）个人类奖项入围分数线
- AI先锋个人：总分≥80分

### （三）奖项空缺机制
如当期无项目达到入围分数线，该奖项可空缺，不为凑数而评奖。

### （四）同分处理规则
若出现同分，按以下顺序决定优先级：

1. Skill沉淀加分高者优先
2. 价值贡献得分高者优先
3. 可复制性得分高者优先
4. 若仍相同，由评审委员会复议决定

---

## 十、评审组织与打分方法

### （一）评审委员会组成
- 制造负责人
- IT/AI负责人
- 业务代表
- 必要时可增加财务、质量或EHS代表参与专项评审

### （二）打分方式
1. 各评委独立打分
2. 汇总后计算项目最终得分
3. 基础分与Skill加分分别取所有评委平均值，最终得分 = 基础分平均值 + Skill加分平均值，保留1位小数

### （三）分差复核机制
若同一项目评委打分中，最高分与最低分差值超过15分，须进行复核讨论，并在评分表中记录原因。

### （四）回避原则
项目直接参与人、直属主管、直接利益相关人原则上不参与本项目最终表决。

---

## 十一、评审流程

### Step 1：项目申报
- 每月统一收集申报项目
- 申报时提交项目材料及证明材料
- 如有Skill卡、SOP、模板、培训材料等沉淀成果，可一并提交参与加分

### Step 2：初筛（AI/IT）
- 剔除非AI项目
- 剔除材料不完整项目
- 核验唯一编号、重复申报情况

### Step 3：材料复核
- 对关键指标进行抽查
- 必要时要求补充截图、报表、视频、现场证明

### Step 4：评审打分
- 采用统一评分表打分
- 重要项目可安排5~10分钟现场答辩

### Step 5：结果公示
- 公示获奖名单、奖项名称、项目名称、负责人
- 公示期内接受异议反馈

### Step 6：复盘沉淀
- 获奖项目需在评优后补齐推广资料
- 纳入制造中心AI案例库与Skill库

---

## 十二、AI Skill卡（推荐沉淀项）

AI Skill卡不再作为参评必备材料，但建议作为重点沉淀项提交。对于高质量Skill卡及其复用成果，可在评审中获得额外加分。

建议统一格式如下：

1. Skill名称
2. 所属模块
3. 解决问题
4. 输入数据
5. 输出结果
6. AI实现方式
7. 适用场景
8. 当前等级（L1-L4）
9. 责任人
10. 可复制说明

说明：Skill卡既是项目沉淀材料，也是后续推广与复用的基础资产。

---

## 十三、奖励机制（建议标准）

| 周期 | 奖励建议 |
| -- | -- |
| 月度 | 500~2000元 |
| 季度 | 3000~10000元 |
| 年度 | 10000元以上 + 晋升参考 |

说明：奖励方式可采用奖金、荣誉证书、公开表彰、晋升参考、专项资源倾斜等组合方式。

---

## 十四、约束机制（防止形式化）

### （一）禁止行为
- 拆分同一项目重复申报
- 使用PPT、方案、概念验证冒充落地项目
- 将传统自动化项目包装成AI项目
- 夸大价值、伪造证明材料

### （二）管控措施
- 同类Skill统一归口管理
- 所有项目必须有唯一编号
- 所有价值项必须提供对应证明材料
- 对弄虚作假项目，取消当期及下一周期参评资格

---

## 十五、制度核心一句话

通过“评优激励 + Skill沉淀 + 规模化推广”，推动制造中心AI能力持续积累，实现效率、质量与组织能力的系统性提升。

---

## 十六、执行建议（落地顺序）

建议按以下顺序启动：

1. 正式发布评优机制
2. 同步下发沉淀材料建议清单（Skill卡、SOP、模板、培训材料等）
3. 在项目管理表中补齐项目编号、责任部门、证明材料链接，并预留Skill沉淀字段
4. 先运行一轮季度评优，验证“基础分 + Skill加分”口径是否需要微调
5. 评优结束后同步沉淀案例库与Skill库
